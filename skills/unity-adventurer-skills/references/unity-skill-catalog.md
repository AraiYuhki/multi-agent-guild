# Unity Adventurer Skill Catalog

このカタログは、冒険者ギルドが Unity 開発を実務で再利用するための標準 Skill 定義です。
各 Skill は **目的 / 想定入力 / 想定出力 / 利用シーン / 適したモデル特性** を必ず含みます。

---

## 🔧 Unity / C# 基盤

### CreateUnityScript
- **目的**: Unity/C# の新規スクリプトを設計・作成し、仕様に沿って実装する。
- **想定入力**: 仕様要件、Unity バージョン、対象コンポーネント、依存 API。
- **想定出力**: `MonoBehaviour` または `ScriptableObject` の実装案、必要なシリアライズ項目、使用例。
- **利用シーン**: 新機能の初期実装、PoC 構築、既存アセットへの統合。
- **適したモデル特性**: **コード特化**（正確な C# 記述、Unity API の堅牢性重視）。

### RefactorUnityCode
- **目的**: 既存 Unity/C# コードを可読性・保守性・パフォーマンス観点で改善する。
- **想定入力**: 対象コード、問題点、制約、改善目標。
- **想定出力**: リファクタリング提案、リスク評価、変更後コード案、検証ポイント。
- **利用シーン**: 技術負債の解消、パフォーマンス改善、拡張性向上。
- **適したモデル特性**: **推論重視**（設計判断 + 影響範囲の把握）。

### AnalyzeUnityErrorLog
- **目的**: Unity コンソール/ビルドログを解析し、原因と対処策を特定する。
- **想定入力**: エラーログ、再現手順、Unity バージョン。
- **想定出力**: 原因分析、修正手順、再発防止策、関連 API。
- **利用シーン**: ビルド失敗、実行時クラッシュ、警告の整理。
- **適したモデル特性**: **推論重視**（根本原因の絞り込み）。

## 🎮 ゲームロジック

### DesignGameStateMachine
- **目的**: ゲーム状態遷移を明確に設計し、状態管理戦略を提示する。
- **想定入力**: ゲームの状態一覧、遷移条件、UI 連携要件。
- **想定出力**: 状態遷移図、`State` クラス設計案、イベント設計。
- **利用シーン**: タイトル/プレイ/ポーズ/リザルト等の設計。
- **適したモデル特性**: **推論重視**（設計と整合性の確保）。

### ImplementPlayerController
- **目的**: プレイヤー操作（移動/ジャンプ/攻撃など）の実装を行う。
- **想定入力**: 操作仕様、Input System/旧 Input、移動方式、物理設定。
- **想定出力**: `PlayerController` 実装案、設定項目、入力マッピング。
- **利用シーン**: 操作系の新規実装、挙動調整。
- **適したモデル特性**: **コード特化**（挙動の正確性重視）。

### ImplementEnemyAI
- **目的**: 敵 AI の行動設計と実装（追跡/巡回/攻撃）を行う。
- **想定入力**: 行動パターン、検知範囲、ナビ/パス仕様。
- **想定出力**: AI ステート設計、行動フロー、実装例。
- **利用シーン**: NPC/敵キャラの挙動設計。
- **適したモデル特性**: **推論重視 + コード特化**（設計 + 実装の両立）。

### BalanceGameParameters
- **目的**: ゲームバランス（数値調整）を設計し、調整方針を提案する。
- **想定入力**: 既存パラメータ、プレイ結果、目標体験。
- **想定出力**: 調整方針、変更候補、試験計画。
- **利用シーン**: 難易度調整、報酬設計、曲線最適化。
- **適したモデル特性**: **推論重視**（体験設計・因果関係の整理）。

## 🎨 UI / UX

### DesignUnityUI
- **目的**: Unity UI の構成を設計し、画面構造と遷移を整理する。
- **想定入力**: UI 要件、画面一覧、操作フロー。
- **想定出力**: UI 階層案、レイアウト方針、必要な Prefab 一覧。
- **利用シーン**: 画面設計、UI 再構築、UX 改善。
- **適したモデル特性**: **推論重視**（情報設計/導線設計）。

### GenerateUIPrefabStructure
- **目的**: UI の Prefab 階層構成を提示し、再利用しやすい構造を提案する。
- **想定入力**: UI 画面仕様、再利用方針、アセット要件。
- **想定出力**: Prefab 階層構造、命名規則、配置ガイド。
- **利用シーン**: UI 実装前の構造設計、再利用ルール整備。
- **適したモデル特性**: **推論重視**（構造整理と再利用性重視）。

## ⚙ プロジェクト改善

### ReviewUnityProjectStructure
- **目的**: Unity プロジェクト構成の品質を評価し、改善案を提示する。
- **想定入力**: フォルダ構成、アセット配置、依存関係。
- **想定出力**: 改善ポイント、リスク、移行ステップ。
- **利用シーン**: プロジェクト整理、チーム運用改善。
- **適したモデル特性**: **推論重視**（設計・運用の整合性）。

### SuggestAssetOrganization
- **目的**: アセットの整理方針を提案し、命名・分類ルールを定義する。
- **想定入力**: 現状のアセット構成、チームの運用ルール。
- **想定出力**: フォルダ設計、命名規則、移行手順。
- **利用シーン**: アセット増大による混乱の解消。
- **適したモデル特性**: **推論重視**（運用ルールの整理）。

### OptimizeUnityPerformance
- **目的**: Unity のパフォーマンス最適化案を提示する。
- **想定入力**: プロファイラ結果、対象プラットフォーム、FPS 目標。
- **想定出力**: ボトルネック分析、最適化手順、検証方法。
- **利用シーン**: FPS 低下、メモリ圧迫、GC スパイク。
- **適したモデル特性**: **推論重視**（分析と優先度整理）。

## 📦 ビルド・運用

### PrepareUnityBuildSettings
- **目的**: Unity のビルド設定を整理し、環境別の構成を提示する。
- **想定入力**: 対象プラットフォーム、シーン構成、ビルド手順。
- **想定出力**: Build Settings チェックリスト、設定例、注意点。
- **利用シーン**: リリース前準備、環境切り替え。
- **適したモデル特性**: **高速モデル**（チェックリスト作成と整理）。

### SuggestUnityCI
- **目的**: Unity プロジェクトの CI 構成案を提示する。
- **想定入力**: リポジトリ構成、対象プラットフォーム、CI 制約。
- **想定出力**: CI 流れ、必要なジョブ、注意点。
- **利用シーン**: 自動ビルド/テスト導入。
- **適したモデル特性**: **推論重視**（構成設計とリスク判断）。

## ➕ 任意追加スキル（必要に応じて拡張）

### CreateScriptableObjectConfig
- **目的**: ScriptableObject を用いた設定管理の設計指針を作る。
- **想定入力**: 管理したいデータ構造、用途、参照先。
- **想定出力**: データ構造案、サンプル ScriptableObject 定義。
- **利用シーン**: バランスパラメータやデータ駆動設計。
- **適したモデル特性**: **コード特化**（C# 実装精度重視）。

### AuditUnityDependencies
- **目的**: 依存パッケージの棚卸しとリスク評価を行う。
- **想定入力**: Package Manager 一覧、使用状況。
- **想定出力**: 依存一覧、リスク指摘、整理案。
- **利用シーン**: 長期運用プロジェクトの健全化。
- **適したモデル特性**: **推論重視**（整理と判断）。
